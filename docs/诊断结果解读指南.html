一、诊断报告长什么样

算法入口：diagnose_by_fault_pattern(fault_features, normal_features)

{
  "conclusion": "检测到: 松靴/滑靴脱落",
  "detected_faults": ["松靴/滑靴脱落"],
  "all_results": [
    {
      "fault_type": "松靴/滑靴脱落",
      "is_fault": True,
      "confidence": 2.00,
      "primary_triggered_count": "2/2",
      "primary_features": [
        {
          "feature": "axes.Z.time_domain.kurtosis",
          "fault_value": 21.71,
          "normal_value": 6.30,
          "ratio": 3.45,
          "threshold": 2.0,
          "triggered": True
        }, ...
      ]
    }, ...
  ]
}

怎么快速看结论

  • 先看 conclusion：一句话告诉你检测到了什么
  • 再看 detected_faults：触发了哪些故障类型（可能多个）
  • 然后逐个看 all_results：每种故障的详细判定过程

二、关键字段解读

字段怎么看注意
is_faultTrue=触发，False=未触发最终结论
confidence0~2，越大越确定>1.0 可以放心，<0.5 不可靠
primary_triggered_count"2/2"=全部触发"1/2"=只触发1个，需谨慎
ratio故障值/正常值>阈值才触发，越大越严重
direction"above"=超过阈值触发"below"=低于阈值触发

置信度计算

confidence = (触发的主要特征数 / 主要特征总数) × min(平均比值, 2.0)
场景计算confidence判断
2个全触发，比值3.01.0 × 2.02.0高度可信
2个触发1个，比值2.50.5 × 2.01.0中度可信
1个触发，比值1.21.0 × 1.21.2中度可信
无触发—0不可信

四、多故障同时触发怎么办

场景处理方式
只触发一种直接按该故障处理
两种触发，置信度差距大按置信度高的优先处理
多种触发，置信度接近可能共存（如配流盘+柱塞同时磨损），建议同时检修
全部未触发设备正常（大概率）或未知故障（小概率）

五、常见误判提醒

  • 不要只看一张图：阶次谱突出不等于故障，要结合包络谱/低频FFT确认
  • 不要忽视基线对比：绝对值没有意义，必须和健康数据对比
  • 注意采样率限制：20kHz采样率下，可用频率上限约7.8kHz
  • 转速波动影响：COT在转速稳定时更准，转速波动大时用TOT
  • 多故障共存：实际中往往是多种劣化叠加，不要试图找"唯一故障"

七、特征路径速查表

时域特征

路径含义异常时说明
time_domain.rms均方根值↑ 整体振动能量增大
time_domain.kurtosis峭度↑ 存在尖锐冲击脉冲（松靴最敏感）
time_domain.crest_factor波峰因子↑ 脉冲成分占比增大

频域特征

路径含义异常时说明
freq_domain.band_energies.band_0_100Hz低频带RMS↑ 低频周期力增强（配流盘最敏感）
freq_domain.spectral_centroid频谱重心↑ 能量向高频偏移

阶次域特征

路径含义异常时说明
order_domain.tot_key_orders.7x_fp7阶(f_p)幅值↑ 柱塞通过频率能量增大
order_domain.tot_key_orders.14x_2fp14阶(2f_p)幅值↑ 多柱塞问题

冲击域特征

路径含义异常时说明
impulse_domain.angle_domain_kurtosis角域峭度↑ 冲击与转速严格同步（松靴关键指标）
impulse_domain.envelope_bands.*.peak_freq包络谱峰值频率=f_p → 确认柱塞冲击
