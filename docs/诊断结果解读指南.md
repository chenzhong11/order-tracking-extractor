# 诊断结果解读指南

> 本文档对应 `order_tracking_extractor` v7.22 的 `diagnose_by_fault_pattern()` 输出。
> 面向工程诊断人员：拿到诊断报告后，怎么判断结论是否可靠、下一步该做什么。

---

## 一、诊断报告长什么样

算法的入口函数是 `diagnose_by_fault_pattern(fault_features, normal_features)`，输出结构：

```python
{
    "conclusion": "检测到: 松靴/滑靴脱落",
    "detected_faults": ["松靴/滑靴脱落"],
    "all_results": [
        {
            "fault_type": "松靴/滑靴脱落",
            "mechanism": "柱塞球头松动，周期性撞击斜盘，冲击力激发结构共振",
            "is_fault": True,
            "confidence": 1.73,
            "primary_triggered_count": "2/2",
            "primary_features": [...],   # 主判定特征
            "secondary_features": [...], # 辅助判定特征
        },
        { ... },  # 配流盘磨损
        { ... },  # 柱塞-缸孔磨损
        { ... },  # 斜盘磨损
    ]
}
```

### 怎么快速看结论

1. **先看 `conclusion`**：一句话告诉你检测到了什么
2. **再看 `detected_faults`**：触发了哪些故障类型（可能多个同时触发）
3. **然后逐个看 `all_results`**：每种故障的详细判定过程

## 二、单个故障结果怎么看

每个故障结果的结构：

```python
{
    "fault_type": "松靴/滑靴脱落",
    "mechanism": "柱塞球头松动，周期性撞击斜盘，冲击力激发结构共振",
    "is_fault": True,
    "confidence": 1.73,
    "primary_triggered_count": "2/2",
    "primary_features": [
        {
            "feature": "axes.Z.time_domain.kurtosis",  # 特征路径
            "fault_value": 21.71,       # 故障信号的特征值
            "normal_value": 6.30,       # 正常基线的特征值
            "ratio": 3.45,              # 比值 = fault / normal
            "threshold": 2.0,           # 判定阈值
            "direction": "above",       # 判定方向
            "triggered": True           # 是否触发
        },
        ...
    ],
    "secondary_features": [...]
}
```

### 关键字段解读

| 字段 | 怎么看 | 注意事项 |
|------|--------|----------|
| `is_fault` | True=触发该故障，False=未触发 | 最终结论 |
| `confidence` | 0~2 之间，越大越确定 | >1.0 可以比较放心，<0.5 基本不可靠 |
| `primary_triggered_count` | "2/2" 表示2个主要特征全部触发 | "1/2" 表示只有1个触发，需谨慎 |
| `ratio` | 故障值/正常值的比值 | >阈值才触发，越大偏离正常越严重 |
| `direction` | "above"=超过阈值才触发，"below"=低于阈值才触发 | 松靴的峭度差是 "below"（负值更大） |

### 置信度怎么算的

```
confidence = (触发的主要特征数 / 主要特征总数) × min(平均比值, 2.0)
```

- 2个主要特征全触发，平均比值 3.0 → confidence = 1.0 × 2.0 = **2.0**（最高）
- 2个主要特征触发1个，比值 2.5 → confidence = 0.5 × 2.0 = **1.0**
- 1个主要特征触发，比值 1.2 → confidence = 1.0 × 1.2 = **1.2**
- 无触发 → confidence = **0**

**经验判断：**
- confidence ≥ 1.5 → 高度可信，基本可以确认
- 1.0 ≤ confidence < 1.5 → 中度可信，建议结合其他证据
- confidence < 1.0 → 低可信度，可能是误报或早期微弱信号

## 三、柱塞泵四种故障的逐项解读

### 故障1：松靴/滑靴脱落（冲击类）

**物理机理：** 柱塞球头松动 → 每转一圈撞击斜盘 → 冲击力激发结构共振

**算法怎么判的：**

| 特征路径 | 物理含义 | 阈值 | 方向 | 看什么 |
|----------|----------|------|------|--------|
| `axes.Z.time_domain.kurtosis` | Z轴峭度 | 2.0x | above | Z轴振动分布是否变得"尖锐"（冲击脉冲） |
| `axes.X.impulse_domain.angle_domain_kurtosis` | X轴角域峭度 | 1.5x | above | 角域中是否有周期性冲击 |
| `axes.X.time_domain.rms` | X轴RMS | 1.5x | above | X轴整体振动能量是否增大 |
| `cross_axis.kurtosis_diff_X_Z` | X-Z峭度差 | -3.0 | below | Z轴峭度是否远高于X轴（冲击方向性） |
| `axes.Z.time_domain.crest_factor` | Z轴波峰因子 | 1.2x | above | 是否有尖锐脉冲叠加 |

**怎么解读结果：**

```
primary_features:
  ✅ Z轴峭度: ratio=3.45, threshold=2.0 → 触发
     → Z轴振动信号出现大量尖锐脉冲，峭度是正常值的3.45倍
     → 说明有周期性冲击叠加在振动信号上

  ✅ X轴角域峭度: ratio=9.86, threshold=1.5 → 触发
     → 等角度重采样后，X轴信号的峭度异常高
     → 冲击与转速严格同步（每转一圈一次），确认是旋转部件问题
```

**松靴的典型特征组合：**
- Z轴峭度飙升（最敏感，通常最先触发）
- X轴角域峭度飙升（确认冲击与转速同步）
- X-Z峭度差为大负值（Z轴冲击远强于X轴，说明冲击沿轴向传播）

**如果只触发了1个主要特征：**
- 可能是早期松靴，冲击还很微弱
- 建议：定期复查，观察峭度趋势

**如果触发了全部特征（primary + secondary）：**
- 确认松靴，且程度较重
- 建议：尽快检修，检查柱塞球头配合间隙

---

### 故障2：配流盘磨损（低频周期类）

**物理机理：** 配流盘端面磨损 → 内泄漏增大 → 流量/压力脉动增大

**算法怎么判的：**

| 特征路径 | 物理含义 | 阈值 | 方向 | 看什么 |
|----------|----------|------|------|--------|
| `axes.X.freq_domain.band_energies.band_0_100Hz` | X轴低频带能量 | 2.0x | above | 低频段（转频谐波区）能量是否增大 |
| `axes.X.order_domain.tot_key_orders.7x_fp` | X轴7阶(f_p)幅值 | 1.5x | above | 柱塞通过频率处幅值是否增大 |
| `axes.X.freq_domain.band_energies.band_100_250Hz` | X轴中频带能量 | 1.5x | above | 中频段能量变化 |
| `axes.X.order_domain.tot_order_mean` | X轴阶次均值 | 1.3x | above | 整体阶次能量是否均匀抬升 |

**怎么解读结果：**

```
primary_features:
  ✅ X轴低频带能量(0-100Hz): ratio=9.57, threshold=2.0 → 触发
     → 低频段能量是正常值的9.57倍！
     → 说明转频及其谐波的能量大幅增加
     → 物理原因：泄漏导致流量脉动，产生低频周期性压力波动

  ✅ X轴7阶(f_p)幅值: ratio=2.3, threshold=1.5 → 触发
     → 柱塞通过频率处幅值增大
     → 配流盘磨损导致每个柱塞口的压力切换更剧烈
```

**配流盘磨损的典型特征组合：**
- 低频带能量大幅增加（最显著特征，比值通常 >5x）
- 7阶(f_p)幅值增大（但不如松靴那么突出）
- 阶次均值整体抬升（不是某个阶次突出，而是"整体胖了"）

**与松靴怎么区分：**
- 松靴：7阶突出 + 峭度飙升（冲击型）
- 配流盘：低频带能量飙升 + 阶次均值抬升（弥散型）
- 关键区别：配流盘磨损的峭度不会像松靴那样飙升

**如果只触发了低频带能量：**
- 可能是配流盘早期磨损
- 建议：监测泄漏量变化趋势

---

### 故障3：柱塞-缸孔磨损（弥散型）

**物理机理：** 柱塞与缸孔间隙增大 → 内泄漏 → 流量脉动 → 柱塞频率谐波增大

**算法怎么判的：**

| 特征路径 | 物理含义 | 阈值 | 方向 | 看什么 |
|----------|----------|------|------|--------|
| `axes.X.order_domain.tot_key_orders.7x_fp` | X轴7阶(f_p)幅值 | 1.5x | above | 柱塞通过频率幅值 |
| `axes.X.order_domain.tot_key_orders.14x_2fp` | X轴14阶(2f_p)幅值 | 1.5x | above | 2倍柱塞通过频率幅值 |
| `axes.X.freq_domain.band_energies.band_250_500Hz` | X轴中频带能量 | 1.3x | above | 中频段能量 |
| `axes.X.freq_domain.band_energies.band_100_250Hz` | X轴中低频带能量 | 1.3x | above | 中低频段能量 |

**怎么解读结果：**

```
primary_features:
  ✅ X轴7阶(f_p)幅值: ratio=1.8, threshold=1.5 → 触发
     → 柱塞通过频率幅值增大
     → 每个柱塞经过配流窗口时的压力脉动增大

  ✅ X轴14阶(2f_p)幅值: ratio=2.1, threshold=1.5 → 触发
     → 2倍柱塞通过频率也增大
     → 说明不止一个柱塞有问题，是系统性的间隙增大
```

**柱塞磨损的典型特征组合：**
- 7阶和14阶同时增大（区别于松靴只有7阶突出）
- 中频带能量增大（250-500Hz，泄漏脉动的频带）
- 峭度不会像松靴那样飙升（没有冲击，是弥散型）

**与配流盘磨损怎么区分：**
- 配流盘：低频带(0-100Hz)能量飙升最显著
- 柱塞磨损：7阶+14阶同时突出更明显
- 两者可能共存（配流盘和柱塞副同时磨损）

---

### 故障4：斜盘磨损

**物理机理：** 斜盘摩擦面磨损 → 柱塞运动不平稳 → 轴向力波动

**算法怎么判的：**

| 特征路径 | 物理含义 | 阈值 | 方向 | 看什么 |
|----------|----------|------|------|--------|
| `axes.Z.order_domain.tot_key_orders.7x_fp` | Z轴7阶(f_p)幅值 | 1.3x | above | Z轴柱塞通过频率幅值 |
| `axes.Z.time_domain.rms` | Z轴RMS | 1.3x | above | Z轴整体振动 |
| `axes.Z.freq_domain.band_energies.band_0_100Hz` | Z轴低频带能量 | 1.2x | above | Z轴低频段能量 |

**斜盘磨损的典型特征：**
- Z轴7阶增大（斜盘磨损影响柱塞轴向运动）
- Z轴RMS增大（轴向振动整体增强）
- Z轴低频带增大（轴向力波动的低频分量）

**与松靴怎么区分：**
- 松靴：Z轴峭度飙升（冲击型）
- 斜盘：Z轴RMS和7阶增大，但峭度不那么突出（渐变型）

## 四、综合判定：多种故障同时触发怎么办

### 场景1：只触发一种故障

```
all_results:
  松靴: is_fault=True,  confidence=1.73 ✅
  配流盘: is_fault=False, confidence=0
  柱塞: is_fault=False, confidence=0
  斜盘: is_fault=False, confidence=0

结论：松靴/滑靴脱落，高置信度
```

→ 直接按该故障处理。

### 场景2：两种故障同时触发

```
all_results:
  松靴: is_fault=True,  confidence=1.50 ✅
  配流盘: is_fault=True, confidence=0.85 ✅
  柱塞: is_fault=False, confidence=0
  斜盘: is_fault=False, confidence=0

结论：松靴为主，可能伴有配流盘早期磨损
```

→ 按置信度高的优先处理。低置信度的作为预警，定期复查。

### 场景3：多种故障置信度接近

```
all_results:
  配流盘: is_fault=True, confidence=1.20 ✅
  柱塞: is_fault=True, confidence=1.10 ✅

结论：配流盘和柱塞副同时磨损
```

→ 这种情况很常见，配流盘磨损和柱塞磨损往往伴随发生（同一个磨损过程）。建议同时检修。

### 场景4：全部未触发

```
all_results:
  松靴: is_fault=False
  配流盘: is_fault=False
  柱塞: is_fault=False
  斜盘: is_fault=False

结论：未检测到已知故障模式
```

→ 两种可能：
1. 设备正常（大概率）
2. 存在未知故障类型（小概率，需人工分析原始数据）

## 五、特征路径速查表

算法输出中的 `feature` 字段是特征在特征字典中的路径，对照此表理解含义：

### 时域特征 (`time_domain`)

| 路径 | 含义 | 异常时说明 |
|------|------|-----------|
| `rms` | 均方根值 | ↑ 整体振动能量增大 |
| `peak` | 峰值 | ↑ 出现大幅脉冲 |
| `peak_to_peak` | 峰峰值 | ↑ 振动幅度范围增大 |
| `kurtosis` | 峭度（超额峭度） | ↑ 信号中出现尖锐冲击脉冲（松靴最敏感指标） |
| `skewness` | 偏度 | ↑ 信号不对称，可能有单向冲击 |
| `crest_factor` | 波峰因子 | ↑ 脉冲成分占比增大 |
| `impulse_factor` | 冲击因子 | ↑ 瞬态冲击增强 |
| `clearance_factor` | 裕度因子 | ↑ 类似波峰因子 |

### 频域特征 (`freq_domain`)

| 路径 | 含义 | 异常时说明 |
|------|------|-----------|
| `band_energies.band_0_100Hz` | 0~100Hz频带RMS | ↑ 低频周期性成分增强（配流盘磨损最敏感） |
| `band_energies.band_100_250Hz` | 100~250Hz频带RMS | ↑ 中低频能量增大 |
| `band_energies.band_250_500Hz` | 250~500Hz频带RMS | ↑ 中频能量增大 |
| `band_energies.band_500_1000Hz` | 500~1000Hz频带RMS | ↑ 中高频能量增大 |
| `spectral_centroid` | 频谱重心 | ↑ 能量向高频偏移（磨损） |
| `spectral_entropy` | 频谱熵 | ↑ 能量分布更分散（弥散型劣化） |

### 阶次域特征 (`order_domain`)

| 路径 | 含义 | 异常时说明 |
|------|------|-----------|
| `tot_key_orders.7x_fp` | 7阶(f_p)幅值 | ↑ 柱塞通过频率能量增大（冲击/泄漏） |
| `tot_key_orders.14x_2fp` | 14阶(2f_p)幅值 | ↑ 2倍柱塞频率增大（多柱塞问题） |
| `tot_key_orders.21x_3fp` | 21阶(3f_p)幅值 | ↑ 3倍柱塞频率增大（严重泄漏） |
| `tot_order_mean` | 全阶次幅值均值 | ↑ 整体阶次能量抬升（均匀磨损） |
| `tot_order_std` | 全阶次幅值标准差 | ↑ 阶次能量分布变化 |
| `cot_order_amplitudes` | COT各阶次幅值列表 | 与TOT对比验证 |

### 冲击域特征 (`impulse_domain`)

| 路径 | 含义 | 异常时说明 |
|------|------|-----------|
| `angle_domain_kurtosis` | 角域峭度 | ↑ 角域中有周期性冲击（松靴关键指标） |
| `angle_domain_crest_factor` | 角域波峰因子 | ↑ 角域脉冲成分增大 |
| `angle_domain_rms` | 角域RMS | ↑ 角域整体能量增大 |
| `envelope_bands.*.rms` | 各频段包络RMS | ↑ 该频段有冲击激励 |
| `envelope_bands.*.peak_freq` | 包络谱峰值频率 | =f_p → 确认柱塞冲击 |

### 轴间关系 (`cross_axis`)

| 路径 | 含义 | 异常时说明 |
|------|------|-----------|
| `rms_ratio_X_Z` | X/Z轴RMS比 | 偏离1.0 → 振动方向性变化 |
| `kurtosis_diff_X_Z` | X-Z峭度差 | 负大值 → Z轴冲击远强于X轴（松靴特征） |
| `peak_ratio_X_Z` | X/Z轴峰值比 | 偏离 → 脉冲方向性 |

## 六、实操建议

### 拿到报告后的标准动作

1. **看 `conclusion`**：有没有触发故障
2. **看 `confidence`**：可信度多高
3. **看触发的特征**：哪个特征的 `ratio` 最大 → 那就是最显著的异常信号
4. **看 `mechanism`**：物理机理是否与现场情况吻合
5. **交叉验证**：如果可能，看原始阶次谱/包络谱，确认特征值对应的图谱形态

### 常见误判场景

| 场景 | 表现 | 怎么避免 |
|------|------|----------|
| 基线信号本身不健康 | 所有 ratio 都很小 | 确保基线取自确认健康的设备 |
| 转速波动大 | 阶次分析不准 | 用TOT（需要转速信号）而非COT |
| 传感器松动 | 峭度飙升但不是故障 | 检查传感器安装状态 |
| 多故障叠加 | 多个故障同时触发 | 按置信度排序，高置信度的优先处理 |
| 早期微弱故障 | ratio 刚过阈值，confidence 低 | 不急于定论，定期复查看趋势 |

### 趋势分析比单次诊断更重要

单次诊断可能有波动，**看趋势更可靠**：

```
时间 →
峭度:  6.3 → 7.1 → 8.5 → 12.3 → 21.7
        正常   正常   关注    警告    故障
```

建议：定期采集数据，记录每次诊断的 confidence 和关键 ratio，画趋势图。当 confidence 持续上升或 ratio 持续增大时，即使单次未触发（is_fault=False），也应提前预警。
