# 诊断结果解读指南

> 本文档对应 `order_tracking_extractor` v7.23。
> 面向工程诊断人员：拿到诊断报告后，怎么判断结论是否可靠、下一步该做什么。

---

## 一、诊断报告长什么样

算法的入口函数是 `diagnose_by_fault_pattern(fault_features, normal_features)`，输出结构：

```python
{
    "conclusion": "检测到多种可能故障，最可能是: 松靴/滑靴脱落",
    "detected_faults": ["松靴/滑靴脱落", "配流盘磨损", "柱塞-缸孔磨损"],
    "all_results": [
        {
            "fault_type": "松靴/滑靴脱落",
            "mechanism": "柱塞球头松动，周期性撞击斜盘，冲击力激发结构共振",
            "is_fault": True,
            "confidence": 2.0,
            "primary_triggered_count": "2/2",
            "primary_features": [...],
            "secondary_features": [...],
        },
        ...
    ]
}
```

### 怎么快速看结论

1. **先看 `conclusion`**：一句话告诉你检测到了什么
2. **再看 `detected_faults`**：触发了哪些故障类型（可能多个同时触发）
3. **然后逐个看 `all_results`**：每种故障的详细判定过程

## 二、单个故障结果怎么看

```python
{
    "fault_type": "松靴/滑靴脱落",
    "mechanism": "柱塞球头松动，周期性撞击斜盘，冲击力激发结构共振",
    "is_fault": True,
    "confidence": 2.0,
    "primary_triggered_count": "2/2",
    "primary_features": [
        {
            "feature": "axes.Z.time_domain.kurtosis",
            "fault_value": 21.71,
            "normal_value": 6.30,
            "ratio": 3.45,
            "threshold": 2.0,
            "direction": "above",
            "triggered": True
        },
        ...
    ],
}
```

### 关键字段解读

| 字段 | 怎么看 | 注意事项 |
|------|--------|----------|
| `is_fault` | True=触发该故障，False=未触发 | 最终结论 |
| `confidence` | 0~2 之间，越大越确定 | >1.0 可以比较放心 |
| `primary_triggered_count` | "2/2" 表示全部触发 | "1/2" 表示只有1个触发，需谨慎 |
| `ratio` | 故障值/正常值的比值 | >阈值才触发 |
| `direction` | "above"=超过阈值才触发 | 大多数特征用 "above" |

### 置信度怎么算的

```
置信度 = (触发的主特征数 / 总主特征数) × min(平均比值, 2.0)
```

- 2 个主特征全部触发，平均比值 2.5x → 置信度 = 1.0 × 2.0 = 2.0（满分）
- 1 个主特征触发，比值 1.8x → 置信度 = 0.5 × 1.8 = 0.9

## 三、支持的故障类型

| 故障 | 类别 | 主判定特征 | 机理 |
|------|------|-----------|------|
| 松靴/滑靴脱落 | 冲击 | Z轴峭度 + X轴角域峭度 | 柱塞球头松动，周期性撞击斜盘 |
| 配流盘磨损 | 低频周期 | 0~100Hz频带能量 + 7阶次幅值 | 端面磨损，内泄漏增大 |
| 柱塞-缸孔磨损 | 低频周期 | 7阶次 + 14阶次幅值 | 间隙增大，流量脉动加剧 |
| 斜盘磨损 | 低频周期 | 转频谐波 | 斜盘面不平，轴向力波动 |
| 轴承外圈故障 | 冲击 | 峭度 + 高频包络RMS | 外圈点蚀，滚动体冲击 |
| 轴承内圈故障 | 冲击 | 峭度 + 高频包络RMS | 内圈点蚀，转频调制 |
| 气穴溃灭 | 冲击(随机) | 高频底噪 + 谱熵 | 气泡溃灭，宽频冲击 |

## 四、统计阈值模式

除了原有的固定比值阈值（2.0x/1.5x），v7.23 新增了统计阈值模式：

```python
from order_tracking_extractor import calibrate_from_signal, compare_threshold_modes

# 计算统计阈值
thresh_config = calibrate_from_signal(normal_signal, sampling_rate=20000.0)

# 三种模式对比
result = compare_threshold_modes(normal_signal, fault_features, normal_features, 20000.0)
```

| 模式 | 阈值来源 | 特点 |
|------|----------|------|
| `fixed` | 基线值 × 倍数 | 简单直接，缺乏统计依据 |
| `3sigma` | μ + 3σ | 99.7% 置信度，对偏态分布可能过严 |
| `p95` | 95% 分位数 | 对偏态分布更稳健 |

## 五、诊断报告生成

```python
from order_tracking_extractor import generate_report_from_signals

path, summary = generate_report_from_signals(
    normal_signals, fault_signals,
    sampling_rate=20000.0, rotational_freq=24.67, n_pistons=7,
    output_path="diagnostic_report.png",
)
```

报告包含：
- 时域波形对比（峭度/RMS 标注）
- FFT 频谱（非均匀横轴，峰值数值标注）
- 包络谱（共振解调，故障填充对比）
- 故障概率排序
- 诊断依据（触发故障、关键频率、触发特征）
- 设备参数 + 结论摘要

## 六、特征路径速查表

### 时域特征

| 路径 | 含义 |
|------|------|
| `axes.X.time_domain.rms` | 均方根值 |
| `axes.X.time_domain.peak` | 峰值 |
| `axes.X.time_domain.kurtosis` | 峭度（超额峭度） |
| `axes.X.time_domain.crest_factor` | 波峰因子 |
| `axes.X.time_domain.skewness` | 偏度 |

### 频域特征

| 路径 | 含义 |
|------|------|
| `axes.X.freq_domain.band_energies.band_0_100Hz` | 0~100Hz 频带 RMS |

| `axes.X.freq_domain.spectral_centroid` | 频谱质心 |

| 路径 | 含义 |

| 路径 | 含义 |

| 路径 | 含义 |

## 七、多故障同时触发怎么办

当多种故障同时触发时：

1. **看置信度排序**：置信度最高的最可能
2. **看机理是否冲突**：松靴（冲击）和配流盘（周期力）可以同时存在
3. **看触发特征是否重叠**：如果两种故障用同一个特征触发，可能是同一种故障的不同表现
4. **结合工况判断**：是否有异常噪声、压力波动、温度升高等辅助信息

## 八、常见误判场景

| 场景 | 原因 | 解决 |
|------|------|------|
| 正常信号触发松靴 | 峭度基线偏高 | 用更多正常样本校准统计阈值 |
| 故障信号未触发 | 特征差异不够大 | 检查采样率是否满足高频共振带要求 |
| 多种故障同时触发 | 特征重叠 | 看置信度排序，结合工况判断 |
| 3σ/P95 模式只检出松靴 | 统计阈值过严 | 增加正常样本量，或用 P95 替代 3σ |
