## 一、中文字体问题

### 1.1 核心矛盾

matplotlib 默认字体（DejaVu Sans）不包含中文字符，渲染时会出现方块或空白。

### 1.2 解决方案

**不要用 `plt.rcParams` 的 `font.sans-serif` 列表去猜字体名**，直接用文件路径注册：

```python
import matplotlib.font_manager as fm

# 优先级：WenQuanYi > Noto Sans CJK
_cjk_path = None
for candidate in [
    "/usr/share/fonts/truetype/wqy/wqy-microhei.ttc",
    "/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc",
]:
    if os.path.exists(candidate):
        _cjk_path = candidate
        break

if _cjk_path:
    fp = fm.FontProperties(fname=_cjk_path)
    font_name = fp.get_name()  # 拿到注册后的字体名
else:
    font_name = "DejaVu Sans"

plt.rcParams.update({
    "font.sans-serif": [font_name, "DejaVu Sans"],
})
```

### 1.3 公式渲染

- **数学公式**用 mathtext：`$f_r$`、`$7f_r$`、`$f_p$`
- **中文文字**直接写普通字符串，不要套在公式里
- 混合写法：`f"$7f_r$ (=$f_p$)"` — 公式部分用 `$`，中文部分不用

### 1.4 踩坑记录

| 坑 | 现象 | 解决 |
|---|------|------|
| `fontfamily="monospace"` | 中文全部变方块 | 去掉，用默认字体 |
| `font.sans-serif` 列表猜名 | 某些环境找不到字体 | 改用文件路径注册 |
| 字体缓存 | 换字体后不生效 | `rm -rf ~/.cache/matplotlib` |
| Noto CJK 字体族名 | 注册后显示为 "Noto Sans CJK JP" 而非 SC | 用 `FontProperties(fname=...)` 强制指定 |
| `fontweight="bold"` | 某些 CJK 字体没有 bold 变体，报 Warning | 可忽略，不影响渲染 |

## 二、布局设计原则

### 2.1 信息分层

```
┌──────────────┬──────────┐
│  时域波形     │ 故障排序  │   ← 第一层：概览
├──────────────┴──────────┤
│     FFT 频谱（全宽）     │   ← 第二层：核心频谱
├──────────────┬──────────┤
│   包络谱      │ 诊断依据  │   ← 第三层：细节+依据
├──────────────┴──────────┤
│       结论摘要           │   ← 第四层：结论
└─────────────────────────┘
```

### 2.2 布局要点

- **全宽图**放中间（FFT 频谱），因为它是核心信息
- **对比图**放同一行（时域+排序，包络谱+依据）
- **结论**放最底部，通栏
- `height_ratios` 控制各行高度，信息密度高的行给更多空间
- `wspace`/`hspace` 控制间距，不要太紧（0.25~0.35 合适）

### 2.3 对比图必须共享坐标系

**错误做法**：正常和故障分两个子图，各自有自己的 y 轴范围

**正确做法**：叠加在同一 axes 上，共享 x/y 轴，直接对比

```python
ax.plot(t, normal_signal, color=C_N, label="正常")
ax.plot(t, fault_signal, color=C_F, label="故障")
```

## 三、非均匀横轴（频谱分段）

### 3.1 问题

频谱有效特征集中在低频（0~600Hz），高频（600~8kHz）基本空白。等间距坐标轴导致低频被压缩、高频浪费版面。

### 3.2 方案：分段坐标变换

定义物理频率区间和视觉宽度权重：

```python
class SegmentedAxis:
    def __init__(self, segments, weights):
        # segments: [(0, 600), (600, 8000)]
        # weights:  [0.70, 0.30]  → 低频占70%版面，高频占30%
        ...
    def transform(self, freqs):
        # 物理频率 → 画布坐标 [0, 1]
        ...
```

### 3.3 使用方式

```python
seg = SegmentedAxis([(0, 600), (600, 8000)], [0.70, 0.30])
canvas_x = seg.transform(freqs)  # 变换后的 x 坐标
ax.plot(canvas_x, magnitudes)
seg.setup_axis(ax, step_per_segment=[50, 1000])  # 低频每50Hz一个刻度，高频每1kHz
```

### 3.4 分段边界标记

在两段交界处画虚线，让读者知道"这里跳了一段"：

```python
for c0, c1 in self.bounds[1:]:
    ax.axvline(c0, color="gray", linestyle=":", lw=0.8, alpha=0.5)
```

### 3.5 刻度标签处理

- 低频（<1000Hz）：显示整数，如 `50`, `100`, `150`
- 高频（≥1000Hz）：显示带 k，如 `1.0k`, `2.0k`

## 四、峰值标注技巧

### 4.1 标注位置

频率标签在图顶部，峰值数值紧跟其下方，蓝色（正常）在上、红色（故障）在下：

```python
# 频率标签
ax.annotate("$7f_r$", xy=(c, y_top * 0.93), ...)
# 正常值
ax.text(c, y_top * 0.87, f"n={val_n:.4f}", color=C_N, fontsize=7, ha="center")
# 故障值
ax.text(c, y_top * 0.81, f"f={val_f:.4f}", color=C_F, fontsize=7, ha="center")
```

### 4.2 数值格式

- 幅值 < 0.01 时用 4 位小数：`f"{val:.4f}"`
- 幅值 ≥ 0.01 时用 3 位小数：`f"{val:.3f}"`

### 4.3 幅值比标注

在关键频率处（如 $f_p$）用箭头+文字框标注故障/正常比值：

```python
ax.annotate(f"$7f_r$ 幅值比: {ratio:.1f}x", xy=(c_fp, amp_fault),
            arrowprops=dict(arrowstyle="->", color=C_FP, lw=2),
            bbox=dict(boxstyle="round,pad=0.4", facecolor="#FFF3E0", ...))
```

## 五、视觉区分度

### 5.1 包络谱正常/故障区分

**问题**：两条线颜色太近，糊在一起。

**方案**：故障信号用半透明填充（`fill_between`），正常信号用实线叠加：

```python
# 先画故障填充（底层）
ax.fill_between(x, 0, fault_mag, color=C_F, alpha=0.15, label="故障 (填充)")
ax.plot(x, fault_mag, color=C_F, lw=1.0, alpha=0.85)
# 再画正常实线（上层）
ax.plot(x, normal_mag, color=C_N, lw=1.2, alpha=0.9, zorder=5)
```

### 5.2 频率高亮

诊断中用到的频率，用半透明竖条高亮：

```python
for freq, label, color in basis["key_frequencies"]:
    c = seg_axis.transform(np.array([freq]))[0]
    ax.axvspan(c - 0.015, c + 0.015, alpha=0.12, color="#00838F")
```

### 5.3 配色方案

```python
C_N    = "#1565C0"   # 蓝 — 正常
C_F    = "#C62828"   # 红 — 故障
C_FR   = "#2E7D32"   # 绿 — 转频
C_FP   = "#E65100"   # 橙 — 柱塞频率
C_HARM = "#6A1B9A"   # 紫 — 谐波
C_DIAG = "#00838F"   # 青 — 诊断依据
```

## 六、诊断依据展示

### 6.1 内容

诊断依据面板应包含：
1. 触发了哪些故障
2. 关键频率（$f_r$, $7f_r$, $14f_r$...）
3. 触发的具体特征路径和比值

### 6.2 排版

- 独立面板，不与其他图重叠
- 用 `ax.axis("off")` 隐藏坐标轴
- 文字框用 `bbox=dict(boxstyle="round,pad=0.5", facecolor="#E0F7FA", ...)`

### 6.3 不要和排序图合并

**踩坑**：把诊断依据文字放在排序图下方（用 `ax.text` 的负 y 坐标），会被下面的图遮挡。

**正确做法**：给诊断依据一个独立的 `subplot`，用 `GridSpec` 分配空间。

## 七、常见问题速查

| 问题 | 原因 | 解决 |
|------|------|------|
| 中文显示方块 | 字体不支持中文 | 用 WQY 或 Noto CJK 字体 |
| 换字体后不生效 | 缓存 | `rm -rf ~/.cache/matplotlib` |
| 字体 bold 报 Warning | CJK 字体无 bold 变体 | 忽略，不影响 |
| 图例被遮挡 | zorder 问题 | 设置 `zorder=5` |
| 刻度标签重叠 | 非均匀轴刻度太密 | 增大 `step_per_segment` |
| 峰值标注和频率标签重叠 | y 坐标太近 | 用固定偏移量错开 |
| 诊断依据被其他图遮挡 | 放在负坐标区域 | 改用独立 subplot |
| `UserWarning: Glyph xxx missing` | 字体不包含该字符 | 检查字体路径是否正确 |
